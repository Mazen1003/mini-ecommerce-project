# Mini E-Commerce Prototype
## Problem Statement
Små lokala butiker har ofta svårt att skapa en enkel digital närvaro där kunder kan se produkter online.

Existerande e-handelssystem är ofta för komplexa och dyra för små verksamheter.

Detta projekt syftar till att modellera och prototypa en enkel e-handelslösning med produkter, kategorier och en varukorg.

## Stakeholders

**Customer**
- Bläddrar produkter
- Ser produktdetaljer
- Lägger produkter i varukorg

**Admin**
- Skapar nya produkter
- Organiserar produkter i kategorier

**Guest**
- Kan se produkter men inte genomföra köp

## Functional Requirements

1. Systemet ska visa en lista med produkter.
2. Användaren ska kunna se detaljer för en produkt.
3. Användaren ska kunna lägga en produkt i varukorgen.
4. Systemet ska visa antal produkter i varukorgen.
5. Admin ska kunna skapa en ny produkt.
6. Produkter ska tillhöra en kategori.

1. ## Non-Functional Requirements

1. Systemet ska använda semantisk HTML (header, nav, main, section, article, footer).
2. Systemet ska vara responsivt och fungera på mobil och desktop.
3. Systemet ska fungera utan externa beroenden.
4. JavaScript ska vara minimal och under 50 rader.
